## 🚀 Estándares de Mensajes de Commit

Para mantener un historial limpio y profesional, utilizamos **Conventional Commits**:

- `feat:` Nuevas funcionalidades (ej: feat: agregar login).
- `fix:` Corrección de errores (ej: fix: error en validación).
- `docs:` Cambios en documentación.
- `style:` Formato, CSS, puntos y comas (sin cambios en lógica).
- `refactor:` Mejora de código que no añade funciones ni arregla bugs.
- `chore:` Tareas de mantenimiento, configuración o herramientas (ej: firebase).
